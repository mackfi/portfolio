Music Web Application

Relevant Course Projects

These are some projects I have created in my academic coursework:

For the sake of academic integrity, the source code of my academic projects are not public. I would still like to discuss the skills exercised in each project along with my thought processes and overall experiences.

Learning Management System (LMS)

This project is actively being worked on in one of my courses. So far, we have dived into the importance of proper HTML/CSS to create semantically meaningful and well-designed web pages. We have also utilized Django, Python, and SQLite to create dynamic web pages that query information from a database and display that information in template pages. We have learned about cloud computing and deployed our website with AWS, and we will soon implement authentication. Deployment available at mackf.info

Sprite Editor

Leveraging the power of C++ and Qt Creator, me and five fellow classmates worked together to bring a feature-rich pixel painting program to life. I was primarily responsible for creating the interactive canvas in addition to other drawing features such as brush sizes and the invert-color brush. Because much of the project was dependent on the ability to draw, I completed my portion early to ensure that other team members would be able to complete their sections and ask questions as needed. Additional features include adding or deleting sprite frames, displaying an animation from multiple sprite frames, and saving/restoring sprite files. By communicating effectively as a team, we fulfilled our ambitious goals.

Spreadsheet

A fully functional spreadsheet capable of evaluating formulas, restoring files, and more. This was my first true test of implementing good software practices, as the duration of this project would span several months. Ensuring good naming conventions, documentation, and using scalable code were all critical throughout the development process.

Chat Server and Client

Allows multiple users to connect to a server and send messages to each other via Transmission Control Protocol. The list of participants can be seen by the user, as well as other users connecting and disconnecting. Users can also send a special command to change their name in the chat. Messages and client connections are logged locally on the device running the server.

Agar.io Client

Emulates the game Agar.io, in which players navigate a 2D world, consuming food and other players to acheive the highest score possible. Handled connection with server using TCP, allowing player connections, player/food positions, and player scores to be updated regularly. All game information was tracked in a relational database using MySQL and SSMS, with webpages querying and displaying this information in real time. Also utilized a logger to track game information locally, which was provided via dependency injection.
